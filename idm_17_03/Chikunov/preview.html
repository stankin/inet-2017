История и процесс создания

Проект решено было сделать в виде игры. Причиной этому был тот факт, что обязанности по созданию игры можно с лёгкостью разделить на всех участников команды. Игру было решено сделать пошаговой, для простоты реализации и усложнения игрового процесса. За идею была взята мини-игра по взлому из следующей игры. Основной принцип игры - оперировать своими параметрами и параметрами противника, для взлома последнего. Для нашего проекта лишние параметры и операции были убраны, оставив только:

  • Здоровье;
  • Защита;
  • Атака.

    • Операции остались следующими:

  • Увеличить защиту;
  • Увеличить атаку, уменьшить защиту;
  • Уменьшить защиту врага;
  • Уменьшить атаку врага;
  • Атака врага.

    • При определении всех параметров и операций были выбраны уже числовые показатели всех характеристик. После определения числовых характеристик была начата работа по написанию логики всех расчётов. Для вызова конкретной операции были написаны обработчики событий, по нажатию клавиш от "1" и до "5". Например, операция повышения защиты:

    if (event.keyCode == 49)
    {
    arm1 += barm;
    }

    У любой игры должна быть цель, цель нашей игры - победить противника. Для этого нужно было создать "электронного оппонента", который представлен обычным алгоритмом.

    function bot(){
    if((dmg2 - arm1) >= 20){
    hp1 = hp1 - (dmg2 - arm1);
    resultHp1.textContent = hp1; if (hp1 <= 0) alert("Вы проиграли. Для повтора обновите страницу");
    if (hp1 <= 0 && hp2 <= 0) alert("Ничья. Для повтора обновите страницу");
    }
    else if ((dmg2 - arm1) <= 10 && (dmg1 - arm2) <= 10) {
    arm2 -= armmin;
    dmg2 += bdmg;
    if (arm2 <= 0) arm2 = 0;
    }
    else if ((dmg1 - arm2) >= 30) {
    arm2 += barm;
    }
    else if (dmg1 - arm2 >= 7) {
    dmg1 -= dmgred;
    if (dmg1 <= 0) dmg1 = 0;
    } else if ((dmg2 - arm1) <= 10 && (dmg2 - arm1) >= 5 && (arm2 - dmg1) <= 10) {
    arm1 -= armred;
    if (arm1 <= 0) arm1 = 0;
    }
    else {
    arm2 -= armmin;
    dmg2 += bdmg;
    if (arm2 <= 0) arm2 = 0;
    }
    }

    Тип игры - пошаговая, а это означает, что первым ходит игрок, а вторым - уже "бот", хотя визуально выглядит как одновременные действия. После написания логики игры и алгоритма противника предстояла самая долгая часть работы - тестирование и баланс. Важно было создать такую игру, чтобы она была "проходима", а сам алгоритм, в свою очередь, должен быть не очень глупым. В результате вышла та игра, в которой алгоритм с первого раза никто не выигрывал (поробуйте, может у Вас получится?).

    var hp1 = hp2 = 150;
    var dmg1 = dmg2 = 35;
    var arm1 = arm2 = 30;
    var bdmg = 9;
    var barm = 8;
    var dmgred = 6;
    var armred = 7;
    var armmin = 10;

    При отладки геймплея необходимо было всё это визуализировать, так как непонятно, какие действия совершает алгоритм, так как по изменениям цифр это понять не только достаточно сложно, но и очень неудобно. Для этого было решено связать анимацию игроков с их действиями, поэтому необходимо было выбрать, какими персонажами игроки являются и где происходят действия. После долгих обсуждений была выбрана тема "вестерн", где в роли игроков - ковбои, которые участвую в дуэли на фоне небольшого городка "Дикого Запада".
    Анимация создавалась вручную из следующего спрайта, с добавлением названия выбранной операции. Но, всё это выглядит "сухо" и мы начали искать подходящие звуки всех действий игроков. Звуком атаки соответственно был выстрел из револьвера, а остальные операции - перезарядка револьвера. Также нашли и добавили фоновый звук. Автоматическое вопроизведение музыки - плохой жест в веб-программировании, но в нашем случае оно полностью оправдано, так как не добавляются лишние клавиши и текст, а также сама мелодия очень тихая и подходящая для атмосферы игры. Для добавления звуков в проект был также использован код js.

    function playSound(url){
    var s = new Audio();
    s.src = url;
    s.play();
    }
