1
Шаг 1. Форк репозитория
На данном этапе нужно сделать простой форк, для этого нужно нажать на кнопку форк правом верхнем углу репозитория и выбрать в модальном окне свой аккаунт
2
Шаг 2. Клонирование репозитория
На данном этапе нужно сделать клон репозитория. Для этого нужно:
  1. Скачать гит из официального источника
  2. Открыть консоль windows (комбинация win + r, в открывшееся поле вбить cmd)
  3. Перейти в нужную дирректорию при помощи команды cd
  4. Открыть форк репозитория и скопировать http ссылку на него
  5. Выполнить команду git clone < ссылка на личный репозиторий >
  6. Перейти в клонированную папку через cd
3
Шаг 3. Создание базовой иерархии и личной страницы
  1. Создать личную папку в локальном репозитории, по пути inet-2017/idm-17-05/<Ваша фамилия>
  2. Внутри папки сделать файл index.html
  3. Внутрь index.html поместить базовый html шаблон
  4. Подключить в шаблон стороннюю библьотеку, например jquery.js
  5. Сохранить файл
  6. В файл database.json, находящийся по пути inet-2017/idm-17-05/database.json добавить ссылку на свой репозиторий по аналогии с другими
4
Шаг 4. Сохранение изменений в репозитории
  1. Перейти в корень репозитория при помощи команды cd
  2. Выполнить команду git status для того чтобы удостовериться что изменния были вынесены
  3. Выполнить команду git add . для добавления изменений в коммит
  4. Выполнить команду git commit -m "Создан личный репозиторий" для того чтобы установить название комита
  5. Выполнить команду git push origin master для отправки изменения в удаленный репозиторий
  6. Далее необходим проверить в удаленном личном репозитории что личная папка и файлы появились в нем
5
Шаг 5. Pull request
После того как мы удостоверились что все файлы находятся на удаленном репозитории нужно сделать pull request в общий репозиторий с которого делали fork. Для этого нужно:
  1. Нажать на кнопку "New pull request"
  2. Выбрать удаленную ветку master, если не выбрана
  3. Нажать на кнопку "Create pull request"
  4. Указать название пула и подтвердить
  5. После чего нужно дождаться поку pull request будет принят
6
Шаг 6. Проверка выполнения
В завершении задания нужно убедиться что все работает корректно, для этого нужно зайти в песочницу своей группы https://stankin.github.io/inet-2017/idm-17-05/members.html и нажать на себя. Если открылась ваша страница, то все выполнено корректно.
